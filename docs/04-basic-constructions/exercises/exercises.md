# Практичні вправи - Тема 4: Базові конструкції

## Вправа 1: Структура тестів

**Завдання:**
Створіть файл з структурою:
- `describe('Додавання задач')` - блок тестів
- 3 тести всередині блоку
- Кожен тест перевіряє різний аспект додавання

**Підказки:**
- Використовуйте `describe()` для групування
- Використовуйте `it()` для окремих тестів

**Рішення:** Дивіться в папці `solutions/`

---

## Вправа 2: Використання context()

**Завдання:**
Створіть структуру з `context()`:
- `describe('Робота з задачами')`
- `context('Коли немає задач')` - тести для порожнього стану
- `context('Коли є задачі')` - тести для стану з задачами

**Підказки:**
- `context()` семантично підкреслює зміну контексту
- Можна використовувати всередині `describe()`

**Рішення:** Дивіться в папці `solutions/`

---

## Вправа 3: Організація тестів

**Завдання:**
Створіть структурований файл тестів:
- Верхній рівень: `describe('TodoMVC')`
- Перший рівень: `describe('Додавання')`, `describe('Відмітка')`, `describe('Видалення')`
- Другий рівень: окремі тести в кожному блоці

**Підказки:**
- Уникайте занадто глибокого вкладення
- Назви мають бути описовими

**Рішення:** Дивіться в папці `solutions/`
